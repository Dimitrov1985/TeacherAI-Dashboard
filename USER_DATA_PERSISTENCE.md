# Система сохранения данных пользователя

## 🎯 Как это работает

Все данные приложения теперь привязаны к **userId** пользователя. Это означает:

### ✅ При логине
- Автоматически загружаются ВСЕ данные этого пользователя из localStorage
- Студенты, оценки, посещаемость, планы уроков, заметки, события — всё загружается

### ✅ При работе с приложением
- Все изменения автоматически сохраняются в localStorage с привязкой к вашему userId
- Не нужно нажимать "Сохранить" — всё сохраняется мгновенно

### ✅ При логауте
- Данные НЕ удаляются из localStorage
- Они остаются на вашем устройстве
- При повторном логине всё вернётся на место

### ✅ Несколько пользователей на одном устройстве
- Каждый пользователь видит только свои данные
- Данные не смешиваются между пользователями
- Можно переключаться между аккаунтами без потери данных

## 📁 Структура хранения

### Формат ключей localStorage:

```
teacher-dashboard:{userId}:students          // Студенты
teacher-dashboard:{userId}:grades            // Оценки
teacher-dashboard:{userId}:attendance        // Посещаемость
teacher-dashboard:{userId}:classes           // Классы
teacher-dashboard:{userId}:subjects          // Предметы
teacher-dashboard:{userId}:periods           // Периоды
teacher-dashboard:{userId}:teachers          // Учителя
teacher-dashboard:{userId}:lesson-plans      // Планы уроков
teacher-dashboard:{userId}:calendar-notes    // Заметки календаря
teacher-dashboard:{userId}:calendar-events   // События календаря
teacher-dashboard:{userId}:teacher           // Профиль учителя
teacher-dashboard:{userId}:materials:{id}    // Материалы урока

journal_{userId}_{classId}_{subjectId}       // Журнал оценок
```

### Пример:
Если userId = "user-123", то студенты хранятся в:
```
teacher-dashboard:user-123:students
```

## 🔄 Миграция данных

При первом запуске после обновления приложение автоматически:

1. **Обнаруживает** старые данные (без userId)
2. **Копирует** их в новый формат (с userId)
3. **Удаляет** старые данные
4. **Отмечает** миграцию как завершённую

Вы ничего не теряете! Все данные сохраняются.

## 💾 Где хранятся данные?

Данные хранятся в **localStorage браузера** на вашем устройстве:

- **Chrome/Edge**: `%LocalAppData%\Google\Chrome\User Data\Default\Local Storage`
- **Firefox**: `%AppData%\Mozilla\Firefox\Profiles\...\storage\default`

### Важно:
- Данные привязаны к домену (localhost:5174 или ваш домен)
- Если очистить данные браузера — данные удалятся
- Если зайти с другого устройства — данных не будет (нужен backend для синхронизации)

## 🚀 Преимущества текущей системы

### ✅ Плюсы:
1. **Быстро** — нет задержек на запросы к серверу
2. **Работает оффлайн** — приложение работает без интернета
3. **Бесплатно** — не нужен сервер и база данных
4. **Приватно** — данные только на вашем устройстве
5. **Не теряются при логауте** — можно безопасно выходить из аккаунта

### ⚠️ Ограничения:
1. **Только на этом устройстве** — данные не синхронизируются между устройствами
2. **Нет резервной копии** — если очистить браузер, данные пропадут
3. **Ограничение объёма** — localStorage имеет лимит ~5-10 МБ

```typescript
  // Пользователь залогинен

```

### Пример использования в store:

```typescript
import { getCurrentUserId } from './auth'

export function loadStudents(): Student[] {
  const userId = getCurrentUserId()
  if (!userId) return []

  const key = `teacher-dashboard:${userId}:students`
  const raw = localStorage.getItem(key)
  return raw ? JSON.parse(raw) : []
}
```

## ❓ FAQ

### Что произойдёт, если я очищу кэш браузера?
Все данные будут удалены. Рекомендуется периодически экспортировать важные данные (например, планы уроков в Word).

### Можно ли использовать приложение на нескольких устройствах?
Да, но данные будут независимыми на каждом устройстве. Для синхронизации нужен backend (Вариант 2).

### Безопасны ли мои данные?
Да, данные хранятся только на вашем устройстве и не передаются на сервер. Но рекомендуется использовать приложение на личном устройстве, не на общедоступном компьютере.

### Что делать при ошибке "Cannot save: No user logged in"?
Это означает, что вы не залогинены. Войдите в аккаунт через страницу логина.

### Сколько данных можно хранить?
localStorage имеет лимит 5-10 МБ в зависимости от браузера. Этого достаточно для:
- ~1000 студентов
- ~10000 оценок
- ~100 планов уроков
- Всех остальных данных

Если превысите лимит, браузер выдаст ошибку "QuotaExceededError".

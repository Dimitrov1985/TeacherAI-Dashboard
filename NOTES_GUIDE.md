# Руководство по работе с заметками календаря

## Обзор функциональности

Система заметок позволяет создавать напоминания и записи для любой даты в календаре. Заметки сохраняются локально в браузере и привязаны к конкретным датам.

## Как использовать

### Просмотр заметок

1. Выберите дату в календаре (кликните на любой день)
2. Справа появится панель "Заметки" с выбранной датой
3. Если на эту дату есть заметки, они отобразятся списком
4. Если заметок нет, появится предложение добавить первую заметку

### Создание новой заметки

#### Способ 1: Через кнопку "+"
1. Выберите дату в календаре
2. В панели "Заметки" нажмите кнопку "+" в правом верхнем углу
3. Откроется модальное окно создания заметки

#### Способ 2: Через пустой список
1. Выберите дату без заметок
2. Нажмите "Добавить первую заметку"

#### Заполнение формы заметки:
- **Заголовок** (обязательно) - краткое название заметки
- **Описание** (опционально) - подробная информация
- **Цвет** - выберите один из 7 цветов для визуального выделения

### Редактирование заметки

1. Кликните на любую существующую заметку
2. Откроется форма редактирования
3. Измените нужные поля
4. Нажмите "Сохранить"

### Удаление заметки

1. Откройте заметку для редактирования
2. Нажмите кнопку "Удалить" в левом нижнем углу
3. Подтвердите удаление

## Визуальные индикаторы

### В календаре:
Под числами месяца отображаются индикаторы:
- **Синяя точка** - событие (из расписания)
- **Жёлтая точка** - есть заметка на эту дату
- Если на дату есть и событие, и заметка - показываются обе точки

### В списке заметок:
- **Цветная полоса слева** - цвет, выбранный для заметки
- **Заголовок** - название заметки
- **Описание** - первые 2 строки (если есть)
- **Дата создания** - время добавления заметки

## Хранение данных

### Где хранятся заметки?
- Заметки сохраняются в `localStorage` браузера
- Ключ хранения: `teacher-dashboard:calendar-notes`
- Данные сохраняются автоматически при создании/изменении

### Безопасность данных
- ✅ Данные хранятся локально на вашем компьютере
- ✅ Не передаются на сервер
- ✅ Доступны только в вашем браузере
- ⚠️ Очистка кеша браузера удалит все заметки
- ⚠️ При использовании другого браузера/компьютера заметки будут недоступны

## Лимиты и ограничения

- Максимальная длина заголовка: без ограничений
- Максимальная длина описания: без ограничений
- Количество заметок на дату: без ограничений
- Общее количество заметок: ограничено размером localStorage (~5-10 МБ)

## Сортировка

Заметки на одну дату сортируются:
- От новых к старым (по времени создания)

## Часто задаваемые вопросы

### Как перенести заметки на другой компьютер?
Пока функция переноса не реализована. В будущем будет добавлен экспорт/импорт.

### Заметки удалились после очистки кеша. Как восстановить?
К сожалению, после очистки localStorage данные невозможно восстановить. Рекомендуется регулярно делать экспорт (когда функция будет добавлена).

### Можно ли добавить напоминание на определённое время?
Пока нет, но функция планируется в будущих обновлениях.

### Сколько заметок можно создать?
Технически без ограничений, но общий размер данных в localStorage ограничен браузером (~5-10 МБ).

## Технические детали

### Структура данных заметки:
```typescript
type CalendarNote = {
  id: string           // Уникальный идентификатор
  date: string         // Дата в формате ISO (yyyy-mm-dd)
  title: string        // Заголовок заметки
  content: string      // Содержимое заметки
  color: string        // Цвет в формате HEX
  createdAt: number    // Время создания (timestamp)
}
```

### API хранилища:
```typescript
loadNotes()                    // Загрузить все заметки
saveNotes(notes)              // Сохранить все заметки
addNote(note)                 // Добавить новую заметку
updateNote(id, updates)       // Обновить заметку
deleteNote(id)                // Удалить заметку
getNotesForDate(date)         // Получить заметки для даты
```

# Руководство по управлению событиями

## Обзор функциональности

Система событий позволяет создавать, редактировать и удалять календарные события. События отображаются в разделе "Upcoming Events" (Предстоящие события) и помогают отслеживать важные даты.

## Как использовать

### Просмотр событий

Компонент "Upcoming Events" показывает:
- События на выбранную дату (если они есть)
- ИЛИ ближайшие 4 предстоящих события (если на выбранную дату нет событий)
- Период отображения: текущая дата + 6 дней

### Создание нового события

#### Способ 1: Через кнопку "+"
1. В панели "Upcoming Events" нажмите кнопку "+" в правом верхнем углу
2. Откроется модальное окно создания события

#### Способ 2: Через пустой список
1. Если событий нет, нажмите "Добавить событие"

#### Заполнение формы события:
- **Название события** (обязательно) - например: "Родительское собрание"
- **Дата** (обязательно) - выберите дату из календаря
- **Время** (обязательно) - формат: ЧЧ:ММ - ЧЧ:ММ (например: 09:00 - 10:00)
- **Цвет** - выберите один из 7 цветов для визуального выделения

### Редактирование события

1. Кликните на любое событие в списке
2. Откроется форма редактирования
3. Измените нужные поля
4. Нажмите "Сохранить"

При наведении на событие появляется иконка редактирования.

### Удаление события

1. Откройте событие для редактирования
2. Нажмите кнопку "Удалить" в левом нижнем углу
3. Подтвердите удаление

## Визуальные элементы

### В карточке события:
- **Цветная полоса слева** (12px) - цвет, выбранный для события
- **Иконка предупреждения** (`IconAlertCircle`) - того же цвета
- **Название события** - основной текст
- **Дата** - день недели, число, месяц (например: "пятница, 3 июля")
- **Время** - с иконкой часов (например: "11:30 - 12:30")
- **Иконка редактирования** - появляется при наведении

### В календаре:
- **Синяя точка** - указывает на день с событием

## Хранение данных

### Где хранятся события?
- События сохраняются в `localStorage` браузера
- Ключ хранения: `teacher-dashboard:calendar-events`
- Данные сохраняются автоматически при создании/изменении

### Первый запуск
При первом запуске приложение автоматически создаёт несколько примеров событий:
- Applied Science Homework
- Technology Exam
- Artificial Intelligence Workshop
- UX Design Conference

Вы можете редактировать или удалить эти примеры.

### Безопасность данных
- ✅ Данные хранятся локально на вашем компьютере
- ✅ Не передаются на сервер
- ✅ Доступны только в вашем браузере
- ⚠️ Очистка кеша браузера удалит все события
- ⚠️ При использовании другого браузера/компьютера события будут недоступны

## Логика отображения

### Приоритет показа:
1. **Если на выбранную дату есть события** - показываются только они
2. **Если на выбранную дату нет событий** - показываются ближайшие 4 предстоящих события

### Сортировка:
События сортируются по дате (от ранних к поздним)

### Период просмотра:
Заголовок показывает диапазон: выбранная дата + 6 дней  
Например: "1 июл. - 7 июл."

## Горячие клавиши

- **ESC** - закрыть модальное окно события

## Лимиты и ограничения

- Максимальная длина названия: без ограничений
- Формат времени: свободный текст (рекомендуется ЧЧ:ММ - ЧЧ:ММ)
- Количество событий: ограничено размером localStorage (~5-10 МБ)
- Отображение в списке: до 4 событий одновременно

## Интеграция с календарём

События интегрированы с мини-календарём:
- **Синяя точка** под датой показывает наличие события
- Клик по дате показывает события этого дня
- События автоматически обновляются при изменении

## Отличия от заметок

### События (Events):
- ✅ Структурированные (название, дата, время, цвет)
- ✅ Показываются в блоке "Upcoming Events"
- ✅ Отображаются с иконкой предупреждения
- ✅ Синяя точка в календаре
- ✅ Для важных дел с конкретным временем

### Заметки (Notes):
- ✅ Свободная форма (заголовок + описание)
- ✅ Показываются в блоке "Заметки"
- ✅ Отображаются как карточки
- ✅ Жёлтая точка в календаре
- ✅ Для общих напоминаний и идей

Вы можете использовать оба типа одновременно!

## Часто задаваемые вопросы

### Можно ли создать событие без времени?
Нет, время обязательно. Но вы можете написать "весь день" или "TBD".

### Как создать событие на несколько дней?
Пока что нужно создать отдельное событие для каждого дня. В будущем добавим поддержку диапазонов дат.

### Можно ли добавить описание к событию?
Пока нет. Для дополнительной информации используйте заметки.

### События синхронизируются между устройствами?
Нет, события хранятся только локально в браузере.

## Технические детали

### Структура данных события:
```typescript
type CalendarEvent = {
  id: string       // Уникальный идентификатор
  title: string    // Название события
  date: string     // Дата в формате ISO (yyyy-mm-dd)
  time: string     // Время (свободный формат)
  color: string    // Цвет в формате HEX
}
```

### API хранилища:
```typescript
loadEvents()              // Загрузить все события
saveEvents(events)        // Сохранить все события
addEvent(event)           // Добавить новое событие
updateEvent(id, updates)  // Обновить событие
deleteEvent(id)           // Удалить событие
getAllEvents()            // Получить все события (отсортированные)
```

### Компоненты:
- `UpcomingEvents.tsx` - список событий
- `EventModal.tsx` - форма создания/редактирования
- `eventsStore.ts` - работа с localStorage
